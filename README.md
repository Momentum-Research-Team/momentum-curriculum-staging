# momentum-curriculum-staging

Pre-release staging repository for Momentum Research Team curriculum development.

## Purpose

This repository is the controlled staging area for developing, reviewing, and organizing data science education materials before publication.

## Repository Layout

- `weekly-materials/`: Week-by-week draft curriculum (`week-01` to `week-12`), each with an `assets/` folder
- `proposals/`: Draft proposals for new curriculum ideas with a shared template
- `resources/`: Shared datasets, templates, and references used during staging

## Staging-to-Main Workflow

Momentum curriculum changes follow this lifecycle:

1. **Drafting**
   - Authors create or update content in `weekly-materials/` or submit new ideas in `proposals/`.
   - Supporting files are stored in local `assets/` folders or shared `resources/` directories.

2. **Peer Review**
   - Team members review structure, pedagogical quality, technical accuracy, and dataset readiness.
   - Feedback is incorporated directly in staging drafts.

3. **Staging**
   - Approved revisions remain in this repository as the pre-release source of truth.
   - Materials are organized, validated, and prepared for final publishing.

4. **Release**
   - Finalized and approved content is promoted from staging into the main curriculum repository.
   - Release-ready materials should be complete, internally reviewed, and aligned with team standards.

## Standards

- Use lowercase kebab-case naming for folders and files where applicable.
- Keep weekly content organized by week number for predictable navigation.
- Store reusable assets in `resources/` and week-specific assets within each week’s `assets/` folder.
